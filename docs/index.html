AI Host Observability

Prometheus-friendly Linux host seam observability for AI, GPU, and RDMA infrastructure, focused on the failures that show up between the GPU, the NIC, PCIe, cgroups, and the host kernel.

This project complements GPU and generic host monitoring by surfacing host-side pressure such as memory PSI, reclaim activity, cgroup memory growth, RDMA registration footprint, mlx5 firmware-page growth, VFIO/IOMMU issues, softirq pressure, and kernel event patterns.

What This Repo Helps Catch

Host Memory Pressure

  • MemAvailable exhaustion
  • memory PSI growth
  • reclaim and swap pressure
  • cgroup memory events

Transport and Device Pressure

  • RDMA and NIC registration growth
  • mlx5 firmware pages
  • softnet drops and IRQ pressure
  • PCIe, VFIO, and IOMMU instability

Recent Operator-Facing Updates

Health Score Now Tracks Real Exported Inputs

The composite host health score is now built from metrics the repo actually emits: GPU availability, filesystem headroom, and InfiniBand port health. That makes the score more trustworthy during live incidents and easier to decompose back to raw signals.

See the health score runbook

Job Stall Detection Now Waits For Sustained No-Progress

Training stall suspicion no longer trips just because a single scrape sees no fresh checkpoint. The heartbeat exporter now tracks whether log progress actually advanced and only marks a stall after sustained missing checkpoint and log progress beyond the configured threshold.

See the stall runbook

In practice, that means the public docs now match the live behavior more closely: the health score is based on real emitted metrics, and “job stalled” means forward progress stopped for long enough to matter operationally.
